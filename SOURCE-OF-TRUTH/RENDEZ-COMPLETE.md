## TWO USER MODES

### 🔴 Mode 1: BOOK FIRST (High Intent)
> "I already booked. Now I need a partner."

**Flow:**
1. User books via REZ (dinner, sports, etc.)
2. Creates plan: "Booked dinner @ 8PM — need 1 person"
3. Others apply
4. User selects
5. Meetup happens

**Why Powerful:**
- Zero timepass users
- High seriousness
- Direct merchant revenue

---

### 🟢 Mode 2: MATCH FIRST (Scale)
> "I want to do something. Let's find someone first."

**Flow:**
1. User creates: "Looking for dinner partner tonight"
2. Others join
3. Match
4. THEN booking via REZ
5. Meetup

**Why Needed:**
- Low friction
- High volume
- Solves cold start

## KARMA SYSTEM

### How Karma Works

| Signal | Karma | Reliability Impact |
|--------|-------|-------------------|
| Shows up (QR verified) | +5 | +0.01 |
| Good feedback | +3 | +0.005 |
| No cancellation | +1 | - |
| No-show | -15 | -0.05 |
| Last-minute cancel | -10 | -0.03 |
| Bad feedback | -5 | -0.02 |
| Reported | -20 | -0.1 |

### Karma Tiers

| Tier | Score | Badge | Privileges |
|------|-------|-------|------------|
| ⭐ Reliable | 80+ | Green | Priority visibility, verified badge |
| 👍 Good | 60-80 | Blue | Normal access |
| ⚠️ New | 30-60 | Gray | Some limits |
| 🚫 Risky | <30 | Red | Restricted |

## MASTER LOOP

```
1. User creates plan (booked or not)
        ↓
2. Others apply/join
        ↓
3. Match happens
        ↓
4. Booking confirmed (if not already)
        ↓
5. QR check-in at venue
        ↓
6. Karma updated (+/-)
        ↓
7. Coins rewarded
        ↓
8. Better matches unlocked
        ↓
9. More plans created → Back to 1
```
