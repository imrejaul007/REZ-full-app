# DETAILED FEATURE SPECS

## F1: Loyalty Program (Genius-Style)

### Features
- **Tiers:** Bronze, Silver, Gold, Platinum
- **Benefits by tier:**
  - Bronze: 5% discount
  - Silver: 10% discount + early access
  - Gold: 15% discount + free breakfast
  - Platinum: 20% discount + room upgrade priority
- **Earn:** 1 coin per ₹100 spent
- **Burn:** 100 coins = ₹1 discount
- **Expiry:** Coins expire after 12 months

### Technical Spec
```
User {
  tier: 'bronze' | 'silver' | 'gold' | 'platinum'
  coins: number
  totalSpent: number
  tierExpires: Date
}

Booking {
  coinsEarned: number
  tierAtBooking: string
  discountApplied: number
}
```

## F2: Channel Manager

### Features
- **Supported Channels:** Booking.com, Expedia, MMT, Goibibo, Agoda, Airbnb
- **Sync:** Real-time inventory, rates, restrictions
- **Mapping:** Room type to channel room ID
- **Parity:** Rate parity checks
- **Performance:** Channel-wise ADR, occupancy

### Technical Spec
```
Channel {
  id: string
  name: string
  credentials: EncryptedJSON
  status: 'active' | 'disconnected'
}

InventorySync {
  channelId: string
  roomTypeId: string
  available: number
  rate: number
  restrictions: JSON
  lastSync: DateTime
}
```

## F3: Dynamic Pricing Engine

### Features
- **Base Rate:** Property-set floor/ceiling
- **Demand Factors:**
  - Occupancy (current + forecast)
  - Lead time (days before check-in)
  - Day of week (weekend premium)
  - Events (local calendar)
  - Season (peak/off-peak)
  - Competition (rate parity)
- **ML Model:** TensorFlow.js for predictions
- **A/B Testing:** Compare strategies

### Pricing Formula
```
Final Rate = Base Rate × Demand Factor × Event Factor × Competitor Factor

Demand Factor:
  < 30% occupancy = 0.85
  30-50% = 1.0
  50-70% = 1.15
  70-90% = 1.30
  > 90% = 1.50

Event Factor:
  Local festival = 1.20
  Conference = 1.25
  Holiday = 1.35
```

## F4: Digital Check-in

### Flow
1. Guest receives "Check-in Ready" notification 24h before
2. Upload ID (passport/Aadhaar/DL)
3. Fill arrival form
4. Accept terms digitally
5. Select room (from available)
6. Payment method for incidentals
7. Get digital key or QR code

### Technical Spec
```
DigitalCheckin {
  bookingId: string
  idUploaded: boolean
  idType: string
  idNumber: string (encrypted)
  formCompleted: boolean
  roomSelected: string
  keyGenerated: boolean
  status: 'pending' | 'in_progress' | 'ready' | 'completed'
}
```
